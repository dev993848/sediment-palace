# Security Policy

## Supported Versions

Security fixes are provided for:
- Latest `main` branch
- Latest release candidate tag

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Use one of the following:
1. Open a private security advisory on GitHub (preferred).
2. Contact maintainers through a private channel if available.

Include:
- Affected version/commit
- Reproduction steps
- Impact assessment
- Suggested mitigation (if known)

## Response Targets
- Initial acknowledgment: within 72 hours
- Triage and severity assignment: within 7 days
- Public disclosure: after fix is available and coordinated
